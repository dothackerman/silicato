# Security Policy

## Supported versions

Dialogos is currently in release-candidate stage.

| Version | Supported |
| --- | --- |
| 0.1.x (RC and final) | Yes |
| < 0.1.0rc1 | No |

## Reporting a vulnerability

Please do not post sensitive vulnerabilities in public issues.

Preferred process:
1. Use GitHub private vulnerability reporting (Security tab, if enabled for this repository).
2. If private reporting is unavailable, contact a maintainer directly through GitHub and share details privately.
3. Use public issues only for non-sensitive hardening suggestions.

Include:
- affected version
- environment details (OS, Python, tmux, Codex CLI)
- reproduction steps or proof of concept
- impact assessment
- suggested mitigation (if available)

## Response expectations

- Initial acknowledgment target: within 3 business days
- Triage outcome target: within 7 business days
- Fix timeline: depends on severity and reproducibility

## Disclosure policy

- We coordinate disclosure after fix availability.
- Credit is given unless the reporter asks to remain anonymous.
